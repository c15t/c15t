## AGENTS.md

Use the c15t CLI to generate an `AGENTS.md` file in your app from the docs bundled in the installed c15t packages.

### Installation

Generate or update the file:

<PackageCommandTabs command="@c15t/cli@rc agents" />

### What it does

- Scans the installed c15t packages in your app
- Reads the packaged docs inside `node_modules/<package>/dist/docs/`
- Writes managed c15t-specific guidance into `AGENTS.md`
- Adds rules for concrete workflows such as consent management, cookie banners, consent dialogs, preference centers, script loading, callbacks, and integrations like Google Tag Manager

### Why use it

The generated `AGENTS.md` is local, version-matched, and package-aware. Agents can read the exact docs that ship with the installed c15t version instead of relying on stale model knowledge.

If your app uses multiple c15t packages, the generated file includes a block for each relevant package.

### Agent philosophy

When an AI tool is helping with c15t behavior, it should read the installed c15t docs first and use model knowledge second. That keeps consent flows, script gating, banner behavior, and integrations aligned with the exact version you have installed.

## Agent Skills

c15t publishes agent skills that give AI coding assistants deep knowledge of c15t's APIs, components, and configuration. Skills are reusable workflows and tool-specific guidance, not version-matched local docs.

### Installation

Via the c15t CLI:

<PackageCommandTabs command="@c15t/cli@rc install-skills" />

Or directly:

<PackageCommandTabs command="skills add c15t/skills" />

### What skills provide

- **Styling customization** — theme presets, CSS variables, component slot styling
- **Internationalization** — translation setup, locale routing integration
- **Script management** — configuring third-party scripts with consent categories
- **Component setup** — ConsentBanner, ConsentDialog, provider configuration

### Supported tools

- Claude Code
- Cursor
- GitHub Copilot (via `.github/skills`)
- Any agent that supports the skills format

***

## When to use which

Use `AGENTS.md` when:

- Your agent can read files in the local project
- You want version-matched docs from the installed c15t packages
- You want concrete guidance for consent management, cookie banners, consent dialogs, preference centers, script loading, callbacks, and integrations

Use agent skills when:

- Your tool supports the skills ecosystem
- You want reusable workflows and tool-specific guidance

Use both when:

- Your tool supports both local file context and skills
- You want local docs plus reusable setup and configuration help
