<section id="intro">
  `ConsentWidget` is a standalone, inline consent management widget. Unlike `ConsentDialog` (which is a modal), the widget embeds directly in your page layout - ideal for privacy settings pages, account preferences, or any page where users should be able to manage consent without a modal overlay.
</section>

<section id="customization">
  ## Configuration

  ```tsx
  <ConsentWidget
    hideBranding
    legalLinks={['privacyPolicy', 'cookiePolicy']}
    noStyle={false}
    disableAnimation={false}
  />
  ```

  ## Accordion Behavior

  Each consent category is rendered as an expandable accordion item. Clicking the category header expands it to show a description and any associated services. Users can toggle individual categories on or off using the switch control. The `necessary` category is always enabled and cannot be toggled.

  ## Styling First

  <Callout type="info">
    Most widget customization should stay in the stock component. Use theme tokens and slots such as `consentWidgetAccordion`, `consentWidgetFooter`, and `toggle` before reaching for compound components. See [Styling Overview](/docs/frameworks/react/styling/overview).
  </Callout>

  ```tsx
  <ConsentManagerProvider
    options={{
      theme: {
        colors: {
          surface: '#fffdf8',
          surfaceHover: '#f6f3ee',
        },
        slots: {
          consentWidgetAccordion: 'rounded-3xl border border-black/10',
          consentWidgetFooter: 'border-t border-black/10 px-6',
          toggle: 'shadow-sm',
        },
      },
    }}
  >
    <ConsentWidget />
  </ConsentManagerProvider>
  ```

  Widget copy should be changed through `ConsentManagerProvider.options.i18n` so the inline UI stays aligned with the rest of the consent experience.

  ## Advanced: Compound Components

  Use compound components only when you need to rearrange the widget's existing primitives:

  ```tsx
  <ConsentWidget.Root>
    <ConsentWidget.Accordion type="multiple">
      <ConsentWidget.AccordionItems />
    </ConsentWidget.Accordion>
    <ConsentWidget.Footer>
      <ConsentWidget.FooterSubGroup>
        <ConsentWidget.RejectButton />
        <ConsentWidget.AcceptAllButton />
      </ConsentWidget.FooterSubGroup>
      <ConsentWidget.SaveButton />
    </ConsentWidget.Footer>
  </ConsentWidget.Root>
  ```

  - `ConsentWidget.Root` тАФ Theme context provider
  - `ConsentWidget.Accordion` тАФ Radix-based accordion root
  - `ConsentWidget.AccordionItems` тАФ Auto-generates toggle items from consent config
  - `ConsentWidget.AccordionItem` тАФ Individual category item
  - `ConsentWidget.AccordionTrigger` тАФ Clickable header for each item
  - `ConsentWidget.AccordionContent` тАФ Collapsible content area
  - `ConsentWidget.AccordionArrow` тАФ Expand/collapse indicator
  - `ConsentWidget.Switch` тАФ Category toggle switch
  - `ConsentWidget.Footer` тАФ Footer container
  - `ConsentWidget.FooterSubGroup` тАФ Groups related buttons
  - `ConsentWidget.AcceptAllButton` тАФ Accepts all consent
  - `ConsentWidget.RejectButton` тАФ Rejects all consent
  - `ConsentWidget.SaveButton` тАФ Saves custom selections

  If the stock widget structure is already correct, stay with tokens and slots instead of rebuilding the layout.
</section>
