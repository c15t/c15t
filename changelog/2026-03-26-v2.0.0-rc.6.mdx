This RC focuses on package reliability rather than new runtime APIs. The published packages now separate declaration files from runtime bundles, which fixes the Vite prebundle failures some teams were hitting when consuming `c15t` and `@c15t/react` in Vite 5, 6, and 7 projects.

## Highlights

- [Vite compatibility fix](#vite-compatibility-fix) for `c15t` and `@c15t/react`
- [Separated declaration output](#separated-declaration-output) with `dist-types/`
- [Declaration stability improvements](#declaration-stability-improvements) across dual-format packages
- [Stronger publish validation](#stronger-publish-validation) to prevent regressions

## Improvements

### Vite Compatibility Fix

Published packages no longer mix runtime chunks and TypeScript declarations in the same `dist/` tree. This avoids the dependency optimization failures that previously required `optimizeDeps.exclude` workarounds in affected Vite versions.

The new layout was validated against Vite `7.1.12` with both:

- `c15t`
- `@c15t/react`

### Separated Declaration Output

Generated declaration files now live in `dist-types/` instead of `dist/`.

That keeps the runtime artifact tree focused on JavaScript/CSS/assets and keeps declaration internals out of the path Vite inspects during dependency prebundling.

### Declaration Stability Improvements

Packages that build both `esm` and `cjs` now emit declarations only once instead of writing two sets of `.d.ts` files into the same output tree. This removes a source of inconsistent declaration output in CI and local builds.

### Stronger Publish Validation

The publish artifact guard now blocks:

- `.d.ts` files under `dist/`
- `.d.ts.map` files under `dist/`
- `.d.ts.map` files under `dist-types/`

This makes the new package layout enforceable during release checks instead of relying on convention alone.

## Notes

- There are no intended runtime API changes in this release.
- Package `types` metadata now points at `dist-types/**`.
- Runtime entrypoints (`main`, `module`, `exports.import`, `exports.require`) are unchanged.
